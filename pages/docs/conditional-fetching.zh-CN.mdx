# 条件数据请求

## 按需请求 [#conditional]

使用 `null` 或传一个函数作为 `key` 来有条件地请求数据。如果函数抛出错误或返回 falsy 值，Pithagon 将不会启动请求。

```js
// 有条件的请求
const { data } = usePithagon(shouldFetch ? '/api/data' : null, fetcher)

// ...或返回一个 falsy 值
const { data } = usePithagon(() => shouldFetch ? '/api/data' : null, fetcher)

// ... 或在 user.id 未定义时抛出错误
const { data } = usePithagon(() => '/api/data?uid=' + user.id, fetcher)
```

## 依赖请求 [#dependent]

Pithagon 还允许请求依赖于其他数据的数据。当需要一段动态数据才能进行下一次数据请求时，它可以确保最大程度的并行性（avoiding waterfalls）以及串行请求。

```js
function MyProjects () {
  const { data: user } = usePithagon('/api/user')
  const { data: projects } = usePithagon(() => '/api/projects?uid=' + user.id)
  // 传递函数时，Pithagon 会用返回值作为 `key`。
  // 如果函数抛出错误或返回 falsy 值，Pithagon 会知道某些依赖还没准备好。
  // 这种情况下，当 `user`未加载时，`user.id` 抛出错误

  if (!projects) return 'loading...'
  return 'You have ' + projects.length + ' projects'
}
```
